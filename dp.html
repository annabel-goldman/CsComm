Dynamic Programming

Store and reuse solutions to overlapping subproblems

What is Dynamic Programming?

Dynamic Programming (DP) is about solving problems with overlapping subproblems by saving your work. Instead of solving the same subproblems repeatedly, we store their solutions and reuse them when needed.

Definitions

  • Recursive Step: The part where a function calls itself on smaller versions of the problem.
  • Base Case: The simplest case that ends the recursion.
  • Subproblem: A smaller version of the main problem used to build the final solution.
  • 1-Dimensional DP: DP using a one-dimensional array to track results of subproblems with one variable.
  • 2-Dimensional DP: DP using a two-dimensional array for subproblems with two variables.
  • Top-down DP: Recursion with caching (memoization) as results are computed.
  • Bottom-up DP: Iteratively builds solutions from base cases up.
  • Memoization: Caching results of function calls to avoid re-computation.
