Dynamic Programming

Store and reuse solutions to overlapping subproblems

What is Dynamic Programming?

Dynamic Programming (DP) is about solving problems with overlapping subproblems by saving your work. Instead of solving the same subproblems repeatedly, we store their solutions and reuse them when needed.

Key Concepts

  • Top-down DP: Uses recursion with memoization to store solutions as we compute them
  • Bottom-up DP: Builds solutions iteratively using tabulation, starting from the base cases

When to Use Dynamic Programming

  • When subproblems overlap - the same subproblem appears multiple times
  • When you can define a recurrence relation - a way to build larger solutions from smaller ones
