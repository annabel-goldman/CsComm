Climbing Stairs Problem

Learn how to solve the Climbing Stairs Problem using dynamic programming. This is a classic problem that teaches us how to optimize space complexity while maintaining efficiency.

The Problem

You're climbing a staircase with n steps. Each time, you can take 1 or 2 steps. Your goal is to figure out how many distinct ways you can reach the top.

Step 1: Start With a Small Example

If n = 5 how many possible paths are there to get to the top?

Start by listing the possibilities manually

Here are all the valid paths:

  • 1 + 1 + 1 + 1 + 1
  • 1 + 1 + 1 + 2
  • 1 + 1 + 2 + 1
  • 1 + 2 + 1 + 1
  • 2 + 1 + 1 + 1
  • 2 + 2 + 1
  • 2 + 1 + 2
  • 1 + 2 + 2

Total: 8 ways

Step 2: Define the Subproblem

Let's think about how you arrive at each step.

To get to step n, where could you have come from?

Could you solve the problem if you already knew how many ways there were to get to step n - 1? What about step n - 2?

That's the key insight: You build the solution for n using the solutions to smaller subproblems.

Subproblem

How many distinct ways can I reach the nth stair if I can take either 1 or two steps each time?

Step 3: Define the Recursive Step

Think through what step you can come from to land on step n.

Recursive Step

To reach step n, you either:

  • Came from step n - 1 (1 step jump)
  • Or from step n - 2 (2 step jump)

ways[n] = ways[n - 1] + ways[n - 2]

Step 4: Define the Base Cases

Every recursive or DP solution needs base cases to stop the recursion or to initialize your table. What are the smallest versions of this problem? How many ways can we get to those stairs?

Base Cases

  • If n = 1: only 1 way (take one step)
  • If n = 2: 2 ways (1+1 or just 2)

Step 5: Try Writing It in Code, Analyze Space and Time Complexity

Python Implementation

def climb_stairs(n):
    if n <= 2:
        return n
    dp = [0] * (n + 1)
    dp[1] = 1
    dp[2] = 2
    for i in range(3, n + 1): 
        dp[i] = dp[i - 1] + dp[i - 2]
    return dp[n]

Complexity Analysis

  • Time complexity: O(n)
  • Space complexity: O(n)

Step 6: Optimize

Think back to what we learned in recursion about memoization or storing past values so we can quickly calculate new values. What else can be wasteful in our code?

Hint: Space optimization means rewriting your solution so it uses less memory, often by avoiding data structures you don't actually need.

Optimization

We're storing the entire dp array, but we only ever use:

  • dp[i - 1]
  • dp[i - 2]

Once we've used dp[i - 3], we don't need it anymore. So instead of keeping an array, we can just keep two variables and update them as we go.

Python Implementation Optimized

def climb_stairs_optimized(n):
    if n <= 2:
        return n
    prev1, prev2 = 1, 2
    for _ in range(3, n + 1):
        current = prev1 + prev2
        prev1, prev2 = prev2, current
    return prev2

Time complexity: O(n)

Space complexity: O(1) Better!!
