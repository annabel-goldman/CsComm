Subset Sum Problem

Learn how to solve the Subset Sum Problem using recursive thinking. This is a classic problem that helps us understand how to break down complex problems into smaller, similar subproblems.

The Problem

You're given a list of integers and a target sum. Your goal is to figure out whether there is a subset of these numbers that adds up exactly to the target.

Step 1: Start With a Small Example

Numbers: [3, 1, 5, 9]

Target Sum: 6

Before jumping to code, take a second and just try to solve it with your intuition: Is there a subset that adds up to 6?

You might try:

  • 3 + 1 = 4 doesn't work
  • 1 + 5 = 6 yes! One and five together make the target sum

Great! You found a working subset. But how did you find it? Again let's remember that algorithms are about being able to repeat a series of actions to reliably find the optimal solution for every input.

Let's Break Your Intuition Down

When you tried 1 + 5, you were making choices about which numbers to include or skip.

How could you repetitively make that choice?

Step 2: Turn the Idea Into a Repeatable Action (Recursive Call)

  • Include the current number, and subtract it from the target
  • Skip it, and leave the target as is
  • So subset_sum(i - 1, target) or subset_sum(i - 1, target - nums[i])

Recursion works by solving smaller versions of the same problem.

Ask yourself:

  • Is it the problem easier with fewer numbers?
  • What happens when you're down to one number?
  • At each step, you make a choice and move one step closer to the base case.

Step 3: Stop When You Hit the Smallest Case (Base Case)

  • If the target hits 0: success!
  • If you run out of numbers: failure (unless that last number equals the target)
if target == 0:
    return True
if i == 0:
    return (nums[0] == target)

Step 4: Try Writing It in Code, Analyze Space and Time Complexity

Step 5: Optimize (With Memoization)

Let's say you already solved subset_sum(3, 4) earlier.

If you hit that again, do you really need to solve it again? Nope.

You can cache the result the first time and reuse it later.

This strategy is called memoization (but you don't need the term to use the idea)
